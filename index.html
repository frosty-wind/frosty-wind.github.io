burpsuite的学习之路

发表于 2019-11-26 | 分类于 ctf学习之路

在学习ctf web的途中burpsuite是一款需要掌握的工具，此篇记录我的学习经历

下载

  burpsuite可以在官网中下载，有专业版和社区版，专业版是需要付费的，社区版虽然不需要付费但是有一些功能是被禁用的，所以可以选择在网上下载破解版，这类资源还是挺多的，也有介绍如何使用破解版的相关流程，所以我选择了破解版。

  需要注意的是，使用burpsuite需要java环境，所以还需先配置JDK，网上有些教程给的下载网站不知道为什么与官网的下载网站不一样，虽然也能配置java环境但是并不能打开jar程序，你能注意到你点开的网站和他给的截图不一样，这个时候就需要自己去jdk官网了，那样才能获得正常的页面（当然这并不是说他给的网站打开的不正常，只是不知道为什么下载的东西不完全）。但我也不知道为什么能下载的东西不一样，但是去官网点开才能和他教程给的符号，所以剩下的只需要按照教程走就行。

浏览器配置

  下载完后则需要对浏览器进行设置，让他经过本地代理从而我们能够使用burpsuite抓包，改包。我常用的浏览器是firefox和chrome并没有使用IE,所以我只需要对这两种浏览器进行设置。

firefox的设置

 进行代理服务器设置可以直接选择在网页设置中设置，点击右上角的设置，点击选项，滑到最下，点击网络设置，然后选择手动代理设置，将ip改为127.0.0.1，端口为8080就行；

 但不建议这么做，因为你如果没使用bp(burpsuite简称)，你则又需要将设置改为系统代理，否则无法进行正常的网页浏览，所以我选择使用firefox的插件FoxyProxy，在firefox中按下Ctrl+Shift+A然后搜索FoxyProxy添加插件就行了，然后进入插件的Options功能中点击ADD进行如下配置

  接下来进行证书的配置，在打开代理功能的时候(一定要打开，否则无法进入),输入网址http://burp 进行访问，点击右上角的证书下载，然后进入firfox的设置-选项-隐私与安全-证书-查看证书-导入刚才下载的证书就行了;剩下的就是在你需要的时候打开就行了。

chrome

 与之前firefox一样，选择下载插件，这里就直接附上教程的传送门。与firefox只有一些小差别。

burpsuite的使用(遇到的一些问题)

  因为英语不够好，所以看官方文档比较吃力，中文教程又不够详细，所以学习使用真是一条漫长的道路，我选择一边谷歌搜索一边看b站上的教程，同时记录一下自己遇到的问题

无法拦截到response

  在按照网上的教程，设置了拦截response的选项后点击刷新页面发现已然无法拦截response，这个问题我搜了很久，但是网上没有相对应问题解答，然后非常吃力的看了下官方文档也没发现这类问题的介绍，所以我就自己尝试，在尝试的过程中发现了一个神奇的现象，就是在拦截request后将host乱改一下比如在后面乱加一些子文件，在网页报错后，从新刷新然后就能拦截到response，显示HTML码，这个问题困扰了很久，我就尝试了很多次，也不知道为什么会这样，尝试了很久以后，我怀疑是burpsuite破解版的问题，又在网上下了其他版本的破解版，甚至是官方的社区版，依然如此。一段时候后我怀疑是浏览器的问题，于是我用上了chrome(前面的尝试都是用的firefox)，在进入我测试的那个网站后我欣喜的发现能拦截到response了，就在我断定是浏览器的问题时候我点击了刷新，发现又不能拦截了。但这同时也给了一个思考，为什么我在进这个网站的时候能拦截到，然后刷新的时候就不行了呢，我就想到了一个问题-缓存！对，就是缓存，于是乎我在浏览器中清除了缓存，点击刷新，binggo能拦截到response了，而且两个浏览器都可以，果然是缓存搞的怪，就这样艰难的解决了这个问题。

  总结一下:无法拦截到response可能是你在浏览这个网页时缓存了，所以你在刷新后，直接调用缓存而不需要服务器再次向你发送报文 (应该是这样的，目前http了解的内容较少，可能理解有误) ，所以你只需要删除一下缓存就行，火狐直接在历史里面就行，而且只需要勾选缓存就行了;chrome则按下Ctrl+Shift+Delete然后勾选其中的清楚缓存的图片和文件(其中的看个人意愿是否勾选全部，但如果需要刷新拦截response则需要勾选上诉提到的选项)。

搭建博客的坎坷道路

发表于 2019-11-24 | 分类于 搭建博客 
##                                         搭博客的起因

​ 经过一段时间学习，逐渐适应了 计算机 软工的学习，在网络上看了各种大佬的博客，心血来潮自己也想整个博客来 玩 记录自己的学习，想起以前一位学长也曾建议过写自己的博客 ，于是乎我就用了将近10个小时的时间终于弄好了（主要还是因为自己太菜了，所以用了这么多时间）。

如何搭建

既然有了想法那么就要付出实践了

  1. 选择你想要的搭建方式

    搭建博客主要分为两种，一种是自己搭建站点，另一种则是在直接在博客平台注册一个账号

  • 自己搭建站点

    如果是自己搭建站点呢，可以选择自己买个域名，然后买虚拟主机，然后再使用wordpress当然也可以选择其他的搭建自己博客（如果是国内的主机可能还需要备案），这样搭建的话花钱是必不可少的；也可以选择 github + hexo来搭建站点，这种就是免费的。

  • 直接在博客平台上注册一个账号

    这种方法就 相对 方便了，直接找个博客平台按照流程注册就行了，至于选择哪一个平台就看个人喜好了，比如说CSDN（广告有点多），博客园，掘金，简书等等。

    秉承着多学习东西的想法，我选择了自己搭建站点中的github + hexo，主题选择了比较精简的next主题 ，没有选择搭建自己的网站是因为搭建自己的网站比较费钱（能省就省），这样顺便还能学习一下github的使用，就这样开始了搭建博客之旅

  1. 搭建方法

    在谷歌上搜索GitHub + hexo可以搜索一大堆教程，我选择了其中的一篇，附上传送门。按照上面的教程走就行了，也许有人看了以后会问我为什么用了将近10个小时，主要是因为实际搭建的过程中有很多坑上面并没有介绍，而我只能通过谷歌来搜索解决方案，所以我接下来主要介绍一下我踩的坑。(除了坑以外，还有一些文章上没介绍的关于主题的布置问题，比如tag，categories的设置，title的设置，去掉hexo强力驱动的页脚等等等，这些建议自行谷歌，当然对于那些不看教程也能自己设置的大佬例外)

  1. 踩坑的惨痛经历

    (1) 这个可能只有我这种笨蛋才会犯，当时找了好久自己的博客url。最后才知道就是github pages下面附带的那个网址

    (2)当在git使用代码 hexo s后发现再输入命令没有反应了，发现居然要在git按下ctrl + c退出本地服务器才能继续执行git命令

    (3)进不去localhost:4000/在谷歌上搜了下说是端口占用的原因，然后需要更改端口，最后我才发现，要进入需要在git中执行了hexo s才行，当然在我按下ctrl + c(退出本地服务器)后无法进入。

    (4)配置SSH的时候因为以前老师为了让我们克隆他的课件，就让我们配置过，不过是使用他传给我们的他的SSH密钥，所以无法在.ssh文件夹中申请自己的密钥，当然配置的时候也就失败了，所以我删除了他的密钥再进行配置。

    ​ 除此之外，我在生成密钥的时候还报错了…….can’t established.主要是因为在.ssh文件夹中少了known_hosts文件，在后面的输入中输入y他就可以自行创建了

    (5)（这个是我耽误时间最多的一样）在本地服务器上检查自己的网页是正常的，但是在github上即给别人看的网页中你的所有框架都消失了最后是一片白板

    这里你需要更改一下_config.yml文件中的url和root属性rul改为你blog的网址，root改为你的项目名(相对路径)。

    (PS)其实这个坑还是很简单，谷歌一搜就有，但是有点需要注意这也是为什么我耽误了这么久的原因，当你hexo d上传到你的github后，需要等待一段时间打开自己的博客界面才能恢复正常，我当时就是很傻的上传后直接打开，发现没有任何效果，就疯狂重弄，甚至重新安装我的hexo，现在想想自己真的蠢。

    (6) 目前还没解决的一个大坑，图片无法显示，在typora中正常显示，但是在网页上无法显示，我花了1个多小时在网上寻求方法，但是用了他们提供的方法依旧解决不了，后面有空再研究研究吧 ===================================================================

    手动分割，终于找到了图片无法显示的解决方案，原来是新版本Hexo3不再支持以前的路径方案，所以需采用新的方法(我之前翻的博客都是hexo3.0版本一下)，附上解决方案的传送门,同时学到了以后找解决方案可以先去看看官方文档

    (7) 当然还有很多很多很小的坑我就不详细写了，或则有一些我忘了的。总之，不会的，不知道的去谷歌，你遇到的问题绝大部分人也会遇到，很多人会在自己的博客上写，对于那些自己解决问题，比如自己查阅官方文档的，或者自己发现解决问题的方法的大佬(我在一篇博客里看到有个大佬自己查看http报文发现问题然后解决问题)我是真心佩服。

总结

  搭建博客用了我整整一天的时间，我是从早上8点半(应该是)开始，一直弄到了晚上8点，除去中间吃饭和参加四级口语考试时间应该有将近10个小时(我没有睡午觉)，真的累到爆了。但其实还有很多东西没弄，我本来想加个评论功能的，在众多插件里面最好用的最方便的是valine，但是我实名认证没有成功，支付宝扫脸时说模糊，而且还不能扫第二次，我吐了，其他的评论插件要么是加载太慢，或者弄起来比较复杂，需要增加文件中的配置，我怕我到时候弄了，又会导致博客出错无法显示又得花时间重新弄(消耗不起这个精力了。

  还有很多东西没弄，比如设置一个动态背景，增加点击特效，或者显示浏览量，加入一个头像。其实还是非常想弄的，但是太累了(一整天确实很累,留个坑以后有时间再弄)，而且很怕到时候哪一步出错导致自己的博客页面无法显示(我中途其实尝试过增加一个动态背景特效，但是没成功，还导致界面没打开，最后把哪些东西删掉，才弄好的，不知道是不是又是因为需要上传后等待一段时间才能恢复正常)，等我以后知识储备更丰富了，英语更优秀了，更能折腾了再来弄一下吧，毕竟现在自己在计算机这一个领域里面了解的太少太少了，很多东西都只能看别人的博客来做(比如这个博客就是看别人的教程来做的)，而且做了以后很多东西还不知道原理，为什么这么做就可以了，比如是如何将本地与github连接在一起的，为什么在配置中删几个#就能用了，那里面并没有源码，很多东西还有疑问，计算机这条路很长，但因为热爱就不会觉得无聊。
